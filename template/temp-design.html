Template temporal de pantalla principal

Hub de entrada para simuladores visuales de sistemas operativos

Esta pantalla principal no debe contener la simulacion completa. Su funcion es presentar el proyecto, dejar clara la naturaleza educativa del sistema y ofrecer un menu visual para entrar a simuladores independientes, cada uno alojado en su propio archivo HTML.

Reglas de esta pantalla principal

El hub inicial debe orientar, no saturar. La simulacion real vive dentro de vistas independientes.

home sin sobrecarga

No mezclar todos los modulos aqui

La pantalla principal no debe mostrar CPU, Gantt, memoria y reemplazo al mismo tiempo. Solo presenta el sistema y el menu de entrada.

Identidad visual consistente

El fondo oscuro, acentos brillantes, tipografia tecnica y lenguaje visual deben repetirse en todos los simuladores para que el proyecto se sienta unificado.

Archivos separados por simulador

Cada modulo debe vivir en su propio HTML para facilitar edicion por secciones, versionado y trabajo paralelo dentro del repositorio.

Estructura temporal sugerida

Estos nombres pueden cambiar despues, pero la separacion por archivo debe mantenerse.
